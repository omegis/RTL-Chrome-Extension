Rotem Daily RTL - Privacy Policy

Last Updated: March 31, 2026 | Version 2.6.0

Summary: Rotem Daily RTL does not collect, store, or transmit any personal user data. The extension operates locally in your browser with one optional external request described below.

1. Information Collection

Rotem Daily RTL does NOT collect any personal information.

The extension does not:

  • Collect browsing history
  • Track user behavior
  • Access website content or user input
  • Store passwords, credentials, or personal data
  • Use cookies or tracking technologies
  • Send any user data to external servers

2. Data Storage

The extension uses Chrome's local storage API to save the following preferences:

  • Extension Enable/Disable State - A boolean (true/false) that remembers if you want the extension active or disabled.
  • Hebrew Font Enabled - A boolean that remembers if the optional Hebrew font feature is active (disabled by default).
  • Selected Font - The name of the Hebrew font you selected (e.g., "Heebo", "Rubik").

These preferences are stored locally on your device only and are never transmitted anywhere.

3. External Network Requests

The extension makes no network requests by default. However, if you choose to enable the optional Hebrew Font feature, the extension loads font files from Google Fonts (fonts.googleapis.com).

  • This request is made solely to load the selected Hebrew font for display purposes.
  • No personal data, identifiers, or browsing information is sent to Google.
  • Google Fonts is a widely trusted, free service. Google's privacy policy for this service is available at Google Fonts Privacy FAQ.
  • If you prefer not to make any external requests, simply leave the Hebrew Font feature disabled (which is the default).

4. Permissions Explanation

The extension requires the following permissions for its core functionality:

Host Permissions (Specific Websites)

  • notion.so - To apply RTL styling to Hebrew text in Notion
  • claude.ai - To apply RTL styling to Hebrew chat messages
  • gemini.google.com - To apply RTL styling in Gemini Canvas
  • dash.bunny.net - To apply RTL styling in Bunny.net forms
  • app.manychat.com - To apply RTL styling in ManyChat builders
  • creators.spotify.com - To apply RTL styling in Spotify Creators comments

What the extension does: Only modifies CSS properties (direction, text-align, padding, font-family) to display Hebrew text correctly. Does not read, modify, or access any page content or user data.

Storage Permission

Used only to save your preferences (enable/disable state, font selection) locally.

Tabs Permission

Used only to detect which website you're currently on (to apply the correct styling) and to communicate settings changes to open tabs. Does not access tab content or browsing history.

5. Data Sharing

Rotem Daily RTL does NOT share any data with third parties because it doesn't collect any data in the first place.

  • No analytics services
  • No advertising networks
  • No data transmission of any kind (except the optional Google Fonts request described above)

6. How the Extension Works

The extension operates locally in your browser:

  1. Detects Hebrew characters using Unicode pattern matching
  2. Applies CSS styling (direction: rtl, text-align: right) to elements containing Hebrew
  3. Optionally applies a user-selected Hebrew font from Google Fonts
  4. All processing happens in your browser's memory
  5. No personal data leaves your device

7. Security

The extension is designed with security in mind:

  • Contains no external dependencies (pure JavaScript)
  • Loads no remote code
  • The only optional external request is loading font files from Google Fonts (a trusted CDN)
  • Operates in a sandboxed browser environment
  • Open source for full transparency

8. Children's Privacy

The extension does not collect data from anyone, including children under 13.

9. Changes to This Policy

If we ever change how the extension handles data, we will update this policy and notify users through the Chrome Web Store listing.
